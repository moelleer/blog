# React Day Berlin

Andra dagen på konferensen och vi kände direkt att den här dagen passade oss bättre. Det var mycket fokus på Next och Vercel, bland annat [Tobias Koppers](https://twitter.com/wSokra), skaparen av [Webpack](https://webpack.js.org/) och [Turbopack](https://turbo.build/pack) som jobbar på [Vercel](https://vercel.com/) pratade.

Det blir väldigt tydligt på en sån här konferens hur mycket inflytande Vercel & Next.js har över React i dagsläget, om det är bra eller dåligt är nog upp till var och en. 😌

Skulle man vilja se på konferensen hittar man stream här: [React Day Berlin](https://portal.gitnation.org/events/react-day-berlin-2023/watch).

## Favoritföreläsningar

### React Server Components

Väldigt bra föreläsning av [Tejas Kumar](https://twitter.com/TejasKumar_) om vad RSC är och hur det egentligen fungerar. Föreläsningen bestod av ett praktiskt exempel där han byggde om en helt klientbaserad React-app till att istället använda sig av Server Components.

Hans slutsats blev att han inte rekommenderade någon att göra det själv, utan att istället bara använda sig av Next som har det inbyggt och används som default i alla deras nya projekt. Men helt klart intressant och man fick bättre föreståelse hur det fungerar bakom kulisserna.

### Rethinking Bundling Strategies

En kul föreläsning av [Tobias Koppers](https://twitter.com/wSokra) där han i ett väldigt högt tempo gick igenom hur en Javascriptbundler faktiskt fungerar, vad för saker man har att ta hänsyn till och varför det inte är ett helt enkelt projekt att ta sig an. Vi fick även med oss ett bra citat efter hans föreläsning som sammfattar det bra.

Jag:
>Det är inte så konstigt att det ibland inte fungerar när man ser hur komplext det är.

Widén:
>Det är konstigt att det faktiskt fungerar när man ser hur komplext det är.¨

### Superpowers of Browser's Web API
Bra föreläsning av [Nikola Mitrovic](https://twitter.com/js_svemirko) som belyste bra hur mycket som finns inbyggt i moderna webbläsare idag. Han rekommenderade alla att titta in på [Web API delen](https://developer.mozilla.org/en-US/docs/Web/API) på MDN oftare än man gör för att hitta intressanta saker. Ofta när man ska bygga saker idag är den vanliga vägen att hitta ett lämpligt paket på NPM som löser problemet, men ofta kan det lösas med ett inbyggt API i webbläsaren istället.

På föreläsningen gick han igenom och hade demo på [Interaction Observer API](https://developer.mozilla.org/en-US/docs/Web/API/Intersection_Observer_API), [Screen Wake Lock API](https://developer.mozilla.org/en-US/docs/Web/API/Screen_Wake_Lock_API) och [Background Sync API](https://developer.mozilla.org/en-US/docs/Web/API/Background_Synchronization_API).

Som avslutning ville han även lyfta "Green Javascript" och hänvisade till [Website Carbon Calculator](https://www.websitecarbon.com/) där man kan få en uppskattning på hur stort klimatavtryck ens site har. En bra sak som man borde försöka hålla så lågt som möjligt i den mån det går. 🌍
